## Что это за проект

`bot-matrix` — учебная коллекция «тонких» стартеров echo-ботов: матрица **мессенджер × технология**, раскладка `<мессенджер>/<технология>/`, каждая ячейка самостоятельна. Поведение echo-бота во всех ячейках: текст → эхо; кнопка «Начать» (payload `{"command":"start"}`) → приветствие отдельным хендлером; сообщение без текста → молчание; нет токена → выход с кодом 1 без сети.

## Поднять проект на новой машине

### Требования
- Node ≥ 18.18 (для всех `*/typescript`).
- Python ≥ 3.10 (для всех `*/python`; vkbottle требует ≥3.10).

> Если на машине нет Python ≥3.10 и системный пакетный менеджер не ставит (например старый Xcode не даёт brew собрать) — поставить интерпретатор через **uv** (готовый бинарник, без компилятора):
> ```
> curl -LsSf https://astral.sh/uv/install.sh | sh
> ~/.local/bin/uv python install 3.12
> ```
> Затем в каждой python-ячейке: `uv venv --python 3.12 .venv && uv pip install --python .venv/bin/python -r requirements.txt`.
> ⚠️ В uv-venv нет симлинка `pip` — вызывай `./.venv/bin/python ...`, не `./.venv/bin/pip`.

### Python-ячейка (пример `vk/python`)
```
cd vk/python
python3 -m venv .venv && ./.venv/bin/pip install -r requirements.txt   # либо путь через uv выше
./.venv/bin/python -c "import bot"                 # import-smoke, без traceback
env -u BOT_TOKEN ./.venv/bin/python bot.py; echo $?   # guard: подсказка + код 1, без сети
./.venv/bin/python verify_local.py; echo $?           # мок-харнесс: ✅ PASS, код 0
```

### TS-ячейка (пример `vk/typescript`)
```
cd vk/typescript
npm install
npm run typecheck                       # код 0
env -u BOT_TOKEN npm start; echo $?      # guard: подсказка + код 1
npm run verify:local; echo $?           # мок-харнесс: ✅ PASS, код 0
```

## Живой запуск VK (требует токена сообщества)

1. В VK: своё сообщество → Управление → **Дополнительно → Работа с API**.
   - Вкладка **Long Poll API**: включить, версия **5.199**; **Типы событий** → отметить **«Входящее сообщение»** (`message_new`).
   - Вкладка **Ключи доступа** → «Создать ключ» с правами **«Управление сообществом» + «Сообщения сообщества»**.
   - Отдельно: Настройки → **Сообщения** → включить «Сообщения сообщества».
2. Положить токен в `.env` каждой ячейки (gitignored): скопировать `.env.example` → `.env`, вписать `BOT_TOKEN=vk1.a.…`.
3. `./.venv/bin/python bot.py` (или `npm start`) → написать сообществу в VK, бот ответит эхом.

## Известные ловушки (durable; продублировано из локальной memory, т.к. она не переезжает)

- **Право токена:** `groups.getLongPollServer` (его дёргает Long Poll при старте) требует право **«Управление сообществом»**. Токен только с «Сообщения» → `groups.getById` работает, но старт падает с `error_code 15, error_subcode 1133` («cannot be called with current scopes»). Решение — пересоздать ключ с правом управления.
- **Шум vkbottle 4.9.0:** при необработанной ошибке логгер падает с `AttributeError: 'StyleAdapter' object has no attribute 'opt'` — это баг логгера, маскирующий первопричину; смотреть строку `VK API Error N` выше трейсбека.
- **Платформа VK:** домен API `api.vk.ru` (с 30.09.2025), версия `5.199`, кнопка «Начать» = сообщение с payload-строкой `"{\"command\":\"start\"}"` (матчить по payload, не по тексту).
- **Ловушки мок-харнессов:** vkbottle — в `message` обязательны `"fwd_messages": []` и `version` (int), `client_info` обязателен, `api.API_URL` с хвостом `/method/`, авторизация в query. vk-io — `agent: undefined` (дефолтный https-agent не умеет `http://`), `client_info` обязателен, авторизация в form-теле, `ctx.messagePayload` = распарсенный payload.
